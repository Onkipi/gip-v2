Implementation Roadmap

4-phase enhancement plan grounded in source code audit

Phase 1 — Critical Fixes
Weeks 1–2 · Zero architectural risk
Increase MC iterations to 10,000 (config.js)
Add P50 percentile to simulateProbabilityBand
Refactor ingestion-service.js — parallel providers
Fix impact_score non-determinism (hash-seed baseImpact)
Add server-side /api/whatif endpoint
Phase 2 — Simulation Depth
Weeks 3–5 · No external ML dependency
Multi-step trajectory MC (replace weighted-choice draw)
Fan chart data output (P10–P90 per time step)
K-means cluster analysis on MC trajectories
Replace Markov chain with configurable causal DAG
Multi-path computation: top-5 paths by probability
Phase 3 — Intelligence Quality
Weeks 6–9 · NLP + XAI enhancements
Transformer NLP sidecar (FastAPI + ONNX DistilBERT)
CAMEO event code classification in normalize.js
SHAP-style attribution decomposition in prediction engine
Epistemic / aleatoric uncertainty decomposition
Natural-language explanation generation from attributions
Phase 4 — UI/UX Overhaul
Weeks 10–14 · Front-end upgrades
PredictionPanel: P10/P50/P90 range bar + fan chart
Scenario tab with side-by-side comparison
ADS-B + naval track Leaflet map overlays
Global risk gauge in header bar
D3 DAG visualization for event chains
